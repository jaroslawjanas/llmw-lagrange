# LLMW-Lagrange: LLM Text Watermarking with Lagrange Interpolation

A robust implementation of LLM text watermarking based on Lagrange interpolation over Galois fields. This system embeds multi-bit watermarks in AI-generated text, enabling traceability and authenticity verification even under adversarial manipulation.

## Overview

This project implements the watermarking technique described in academic research, using Lagrange interpolation to embed secret information into text generated by large language models. The system works by:

1. **Encoding**: Biasing token selection during generation to embed points from a secret line equation `f(x) = a₀ + a₁x` over GF(2ⁿ)
2. **Decoding**: Extracting the embedded points from generated text
3. **Verification**: Using Maximum Collinear Points (MCP) algorithms to recover the secret line and verify watermark authenticity

## Key Features

- **Multi-bit Watermarking**: Embeds n-bit blocks using Lagrange interpolation over Galois fields
- **Configurable Parameters**: Adjustable field size, bias strength, and generation settings
- **Model Agnostic**: Works with any HuggingFace transformer model
- **Error Correction**: k-bit error correction variants for robust watermark recovery
- **Statistical Analysis**: Comprehensive tracking and analysis of watermarking effectiveness
- **Flexible Input**: Supports custom prompts or dataset-driven generation

```bash
cd llmw-lagrange
```

2. Create and activate the conda environment:
```bash
conda env create -f environment.yml
```


## Quick Start

### Basic Usage

Generate watermarked text with default settings:

```bash
python main.py --prompt "Write a story about artificial intelligence"
```

### Advanced Configuration

```bash
python main.py \
    --model "microsoft/DialoGPT-medium" \
    --prompt "Explain quantum computing" \
    --max-tokens 256 \
    --n 8 \
    --bias 6.0 \
    --temperature 0.7 \
    --verbose
```

### Batch Processing

Process multiple prompts from a dataset:

```bash
python main.py \
    --dataset "ChristophSchuhmann/essays-with-instructions" "default" "train" "instructions" \
    --n-prompts 10 \
    --max-tokens 304 \
    --stats
```

## Command Line Arguments

### Core Parameters
- `--model`: HuggingFace model identifier (default: "facebook/opt-125m")
- `--max-tokens`: Maximum tokens to generate (must be divisible by n)
- `--n`: Galois field size parameter GF(2ⁿ) and block size (default: 8)
- `--prompt`: Custom text prompt
- `--seed`: Random seed for reproducibility (default: 42)

### Watermarking Parameters
- `--green-fraction`: Fraction of vocabulary in green list (default: 0.5)
- `--bias`: Logit bias for watermark tokens (default: 6.0)

### Generation Parameters
- `--temperature`: Sampling temperature (default: 0.0 for greedy)
- `--context-window`: Maximum context length (default: 1500)
- `--hash-window`: Previous tokens for hashing (default: 1)

### Dataset Parameters
- `--dataset`: HuggingFace dataset specification (4 arguments: name, subset, split, column)
- `--n-prompts`: Number of prompts to process (default: 1)

### System Parameters
- `--cache-dir`: Cache directory for models/datasets
- `--no-cuda`: Disable CUDA acceleration
- `--verbose`: Show detailed output
- `--stats`: Display statistics summary
- `--force-tokenization`: Force detokenization and retokenization for "full simulation" (use with caution)

## Architecture

### Core Components

#### `src/llm_watermark.py`
- **`LLMWatermarkerBase`**: Abstract base class with shared functionality
- **`LLMWatermarkEncoder`**: Implements watermark embedding during text generation
- **`LLMWatermarkDecoder`**: Extracts watermark information from generated text
- **`MCPSolver`**: Solves Maximum Collinear Points problem for verification

#### `src/pm_galois.py`
- **`GaloisField`**: Optimized GF(2ⁿ) implementation with precomputed tables
- **`max_collinear_points`**: Efficient collinear point detection algorithm
- **`recover_line_equation`**: Recovers secret line equation from points

#### `src/model_formatters.py`
- Model-specific prompt formatting using AutoProcessor
- Fallback handling for models without chat templates

#### `src/utils.py` & `src/paths.py`
- Dataset loading and shuffling utilities
- Cache and path management
- HuggingFace token handling

### Watermarking Process

1. **X-coordinate Generation**: Hash previous token ID with secret key to generate x ∈ GF(2ⁿ)
2. **Y-value Calculation**: Compute y = f(x) = a₀ + a₁x using secret line equation
3. **Bit Encoding**: Convert y to n-bit binary representation
4. **Token Biasing**: For each bit, bias vocabulary toward green (bit=1) or red (bit=0) tokens
5. **Block Generation**: Generate n consecutive tokens to encode one y-value

### Verification Process

1. **Decoding**: Extract (x,y) points from generated text by classifying tokens
2. **MCP Solving**: Find maximum set of collinear points using optimized algorithms
3. **Line Recovery**: Recover coefficients a₀, a₁ from collinear points
4. **Verification**: Compare recovered coefficients with original secret values

## Known Issues

**Tokenization Inconsistency**: The `--force-tokenization` flag enables "full simulation" of the method as it would be used in a practical setting, where text is detokenized after generation and retokenized during decoding. However, in some cases (approximately 1 in 20) detokenization does not occur correctly for unknown reasons, causing encoding bits to not match decoded ones. The cause of these inconsistencies is unknown.

**Default Behavior**: By default, the system skips detokenization and uses generated token IDs directly for decoding, which avoids this issue and provides more reliable results.

**Recommendation**: Use `--force-tokenization` with caution and only when you specifically need to simulate the full practical deployment scenario:

```bash
python main.py --force-tokenization --prompt "Your prompt here"
```

## Performance Considerations

- **Memory Usage**: Large models and long generations require significant memory
- **GPU Acceleration**: CUDA support recommended for faster inference
- **Context Window**: Automatic K/V cache trimming prevents memory overflow
- **Error Correction**: Higher k values exponentially increase computation time
